## 1. 제품 개요

### 1.1 제품명
**BebeCare** (베베케어)

### 1.2 한 줄 설명
임신부터 육아까지, AI가 맞춤 조언하는 정보 통합 슈퍼앱

### 1.3 배경 및 문제 정의

#### 현재 문제
1. **정보 파편화**: 임신·출산·육아 정보가 맘카페, 블로그, 정부 사이트 등에 흩어져 있음
2. **맥락 부재**: 사용자 상황(임신 주차, 직장, 지역)을 고려한 맞춤 정보 부족
3. **능동적 안내 부족**: 사용자가 직접 검색해야 하며, 중요 일정을 놓치기 쉬움
4. **신뢰도 문제**: 검증되지 않은 정보 혼재, 출처 불명확

#### 사용자 페인 포인트
- "28주차 직장인 임신부에게 지금 필요한 정보가 뭐지?" (맥락 기반 정보 부족)
- "태아보험은 언제 가입해야 하나? 출산휴가는 언제부터?" (시기별 가이드 부족)
- "정부 지원금, 어린이집 정보를 찾으려면 여러 사이트 돌아다녀야 함" (정보 분산)
- "이 정보가 맞는 건가? 출처가 믿을만한가?" (신뢰도 불안)

## 2. 제품 비전 및 목표

### 2.1 제품 비전
**"모든 부모가 아이를 키우는 여정에서 필요한 정보를 정확한 시기에, 맞춤형으로 제공받는 세상"**

### 2.2 제품 미션
- **통합**: 파편화된 임신·출산·육아 정보를 한 곳에서 제공
- **개인화**: AI 기반으로 사용자 프로필에 맞춘 조언 제공
- **선제적 안내**: 시기별로 필요한 정보를 앱이 먼저 알림
- **신뢰**: 공공 데이터 우선, 출처 명시, 전문성 확보

### 2.3 사업 목표
- **단기 (6개월)**: MVP 출시, 초기 사용자 1,000명 확보
- **중기 (1년)**: MAU 10,000명, 사용자 만족도 4.0/5.0 이상
- **장기 (2년)**: MAU 50,000명, 육아 앱 Top 5 진입, 전문가 상담 서비스 런칭

## 3. 타겟 사용자

### 3.1 Primary Persona: 임신부
- **연령**: 25-40세
- **특성**: 첫 아이 임신 중, 직장인 (60%) vs 비직장인 (40%)
- **핵심 니즈**: 출산 준비, 태아보험, 정부 지원금, 출산휴가 계산

### 3.2 Secondary Persona 1: 신생아 부모 (0-12개월)
- **특성**: 신생아 육아 중, 맞벌이 많음
- **핵심 니즈**: 예방접종, 발달 단계, 이유식, 정부 지원금

### 3.3 Secondary Persona 2: 영유아 부모 (1-5세)
- **특성**: 맞벌이 부부, 어린이집 이용
- **핵심 니즈**: 어린이집 정보, 발달 단계, 교육, 정부 지원금

## 4. 핵심 기능

### 4.1 Epic 1: 사용자 프로필 관리
**목적**: 개인화된 정보 제공을 위한 기초 데이터 수집

**주요 기능**:
1. **온보딩 플로우**
   - 임신 여부 선택 (임신 중 / 육아 중)
   - 임신 주차 또는 출산예정일 입력
   - 자녀 정보 입력 (생년월일, 성별)
   - 직장 여부 입력 (근무 중 / 비근무)
   - 지역 입력 (시·도, 시·군·구)

2. **프로필 관리**
   - 프로필 수정 (임신 주차 자동 업데이트)
   - 자녀 추가/수정/삭제
   - 계정 설정 (알림, 언어 등)

**사용자 스토리**:
- "임신부로서, 나는 임신 주차와 출산예정일을 입력하여 내 상황에 맞는 정보를 받고 싶다"
- "직장인 임신부로서, 출산휴가 정보를 맞춤으로 받고 싶다"

---

### 4.2 Epic 2: AI 기반 맞춤 정보 제공 엔진
**목적**: 사용자 프로필 기반으로 개인화된 조언 제공

**주요 기능**:
1. **AI 챗봇 (대화형)**
   - 사용자 질문에 대한 맞춤 답변
   - 프로필 정보를 프롬프트에 자동 주입
   - 대화 히스토리 저장 및 불러오기

2. **맞춤 조언 카드**
   - 홈 화면에 "지금 알아야 할 정보" 자동 표시
   - 예: "28주차입니다. 태아보험 가입을 서두르세요"

**기술 구현**:
- OpenAI GPT-4o-mini API 직접 호출
- 프롬프트 템플릿에 사용자 프로필 동적 주입
- 대화 히스토리는 Supabase DB에 저장

**사용자 스토리**:
- "임신부로서, '출산휴가는 언제부터 쓸 수 있나요?'라고 물으면 내 출산예정일 기준으로 답변을 받고 싶다"
- "신생아 부모로서, '우리 아기가 목을 못 가누는데 괜찮나요?'라고 물으면 아이 개월 수에 맞는 답변을 받고 싶다"

---

### 4.3 Epic 3: 시기별 체크리스트 및 알림
**목적**: 사용자가 중요 일정을 놓치지 않도록 선제적 안내

**주요 기능**:
1. **타임라인 자동 생성**
   - 임신 주차별 체크리스트 (예: 22주 태아보험, 34주 출산휴가)
   - 육아 개월별 체크리스트 (예: 2개월 예방접종, 6개월 이유식)
   - 완료 여부 체크

2. **알림 (Push Notification)**
   - 중요 일정 1주일 전, 3일 전, 당일 알림
   - 알림 설정 (ON/OFF, 시간대 선택)

3. **홈 화면 타임라인**
   - "이번 주 할 일", "다음 주 할 일" 표시
   - 완료한 항목은 체크 표시

**사용자 스토리**:
- "임신부로서, 태아보험 가입 시기를 놓치지 않도록 미리 알림을 받고 싶다"
- "신생아 부모로서, 예방접종 일정을 자동으로 관리하고 알림 받고 싶다"

---

### 4.4 부가 기능 (MVP 제외, Phase 2 이후)

- 검색 기능
- 즐겨찾기

#### 커뮤니티
- 사용자 간 Q&A
- 경험 공유

## 5. MVP 범위

### 5.1 MVP 포함 기능
✅ **Epic 1: 사용자 프로필 관리**
- 온보딩 (임신 주차, 자녀 정보, 직장, 지역)
- 프로필 수정

✅ **Epic 2: AI 맞춤 조언 엔진**
- AI 챗봇 (대화형, OpenAI API)
- 맞춤 조언 카드 (홈 화면)

✅ **Epic 3: 시기별 타임라인 및 알림**
- 타임라인 자동 생성 (임신 주차별, 육아 개월별)
- 체크리스트 완료 처리
- 푸시 알림

✅ **인증 및 기본 기능**
- 회원가입/로그인 (Supabase Auth)
- 설정 (알림 ON/OFF)

---

### 5.2 MVP 제외 기능 (Phase 2 이후)
❌ **정보 통합 허브**: 최소한의 샘플 데이터만 포함, 본격적인 콘텐츠 브라우징은 Phase 2

## 6. 성공 지표 (KPI)

### 6.1 사용자 획득 지표
- **MAU (Monthly Active Users)**: 1,000명 (3개월 후 목표)
- **신규 가입자 수**: 월 300명 이상
- **온보딩 완료율**: 80% 이상 (회원가입 → 프로필 입력 완료)

### 6.2 사용자 참여 지표
- **DAU/MAU 비율**: 30% 이상 (일일 활성 사용자 비율)
- **AI 챗봇 사용률**: MAU 대비 50% 이상 (최소 1회 이상 사용)
- **타임라인 체크 완료율**: 70% 이상 (생성된 체크리스트 중 완료 비율)

### 6.3 만족도 지표
- **앱 평점**: 4.0/5.0 이상 (App Store, Google Play)
- **NPS (Net Promoter Score)**: 40 이상
- **AI 답변 만족도**: 4.0/5.0 이상

### 6.4 기술 지표
- **앱 로딩 속도**: 2초 이내 (홈 화면 초기 로드)
- **AI 응답 속도**: 5초 이내
- **에러율**: 1% 이하
- **푸시 알림 도달률**: 90% 이상

## 7. 비기능 요구사항

### 7.1 성능
- **페이지 로드 시간**: 2초 이내 (3G 네트워크 기준)
- **AI 응답 시간**: 5초 이내
- **동시 접속자 처리**: 1,000명 (초기 목표)

### 7.2 보안 및 개인정보
- **Supabase RLS (Row Level Security)**: 사용자별 데이터 접근 제어
- **HTTPS**: 모든 통신 암호화 (Vercel 자동 적용)
- **개인정보 처리방침**: 임신 정보, 자녀 정보 등 민감 정보 처리 명시
- **의료 정보 면책**: "참고용이며 전문가 상담 권장" 안내 문구 필수

### 7.3 접근성
- **WCAG 2.1 AA 준수**: 스크린 리더 지원, 키보드 네비게이션
- **다국어 지원**: 1차는 한국어만, 향후 영어 확장

### 7.4 호환성
- **웹 브라우저**: Chrome, Safari, Edge (최신 2개 버전)
- **모바일**: iOS 15+, Android 10+
- **반응형**: 모바일 우선, 태블릿/데스크톱 지원

## 8. 제약사항 및 리스크

### 8.1 제약사항
1. **의료 정보 제공 제한**: 의료법상 진단·처방 불가, "참고용" 명시 필수
2. **데이터 저작권**: 공공 데이터 우선 활용, 출처 명시
3. **AI 답변 정확성**: OpenAI API 의존, 오답 가능성 존재

### 8.2 리스크 관리

#### 기술 리스크
- **OpenAI API 장애**: Fallback 메시지 표시 ("일시적 오류, 잠시 후 다시 시도")
- **Supabase 장애**: 캐싱 전략, 에러 메시지 표시

#### 비즈니스 리스크
- **사용자 획득 실패**: 맘카페 등 기존 커뮤니티 연계, SNS 마케팅
- **콘텐츠 부족**: 초기는 핵심 카테고리만 제공, 점진적 확대

#### 법적 리스크
- **의료 정보 제공 이슈**: "참고용" 명시, 전문가 상담 권장 안내
- **개인정보 보호**: Supabase RLS 적용, 개인정보 처리방침 명시
